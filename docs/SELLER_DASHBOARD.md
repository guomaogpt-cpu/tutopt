# Seller Dashboard (MVP)

Кабинет продавца `/seller/dashboard`. Доступ: SELLER (после онбординга) и ADMIN.
BUYER редиректится на upgrade, guest — на login.

## Блоки

1. **Header** — название компании (или имя пользователя), badge «Продавец», город, дата регистрации, кнопка «Подать объявление». Если trust level `incomplete` — предупреждение «Профиль продавца заполнен не полностью».
2. **Статус аккаунта** — labels из `getUserRestrictionLabels`: Активен / Заблокирован / Ограничено создание объявлений / Ограничены заявки. При ограничениях — «Аккаунт ограничен. Некоторые действия недоступны.» Технические поля (`blocked_at` и т.п.) не показываются.
3. **Карточки статистики** — всего объявлений, активные, на модерации, отклонённые, заявки всего, новые заявки (`LeadStatus.NEW`), направления с объявлениями.
4. **Заполненность профиля** — score/level/improvements из `calculateSellerTrust` с progress bar. Только честные сигналы («Телефон подтверждён», «Профиль заполнен»), без «проверенный продавец».
5. **По направлениям** — количество объявлений по OPT/MARKET/SERVICES/CARGO.
6. **Быстрые действия** — подать объявление, заявки, публичный профиль (`/seller/[slug]`), каталог + create-ссылки по vertical. Кнопки «Настроить профиль» нет: route `/seller/settings` не существует (нерабочие кнопки не добавляем).
7. **Последние заявки** — последние 5: vertical badge, статус, объявление, имя покупателя, дата. Ссылка «Открыть заявки» → `/seller/leads`. Контакты покупателя на дашборде не выводятся (они доступны в `/seller/leads`, как и раньше).
8. **Мои объявления** — существующий блок с табами по статусам; каждая карточка: фото, vertical, статус, цена, просмотры, quality badge (Хорошо / Нужно улучшить / Проблемное) и до 2 подсказок качества.

## Данные и helpers

- `prisma.sellerProfile` + listings (с городом, категорией, фото)
- `prisma.lead` — счётчики и последние 5 заявок
- `calculateSellerTrust` (`src/lib/trust/seller-trust.ts`) — заполненность профиля
- `calculateListingQuality` (`src/lib/moderation/listing-quality.ts`) — quality badges и warnings
- `getUserRestrictionLabels` (`src/lib/security/user-restrictions.ts`) — статус аккаунта

Prisma schema не менялась, миграций нет.

## Ограничения, видимые продавцу

Только человекочитаемые labels. Заблокированный продавец видит предупреждение,
но dashboard остаётся доступным — сами действия (create listing, leads, onboarding)
блокируются на API-уровне (Phase 19).

## Как считается заполненность

`calculateSellerTrust`: score 0–100 из сигналов (телефон подтверждён, профиль заполнен,
описание, город, лого, активные объявления, возраст аккаунта). Level: `trusted` ≥ 80,
`normal` ≥ 50, иначе `incomplete`. Improvements — список конкретных шагов.

## Listing quality hints

`calculateListingQuality` возвращает level (good/warning/bad) и warnings
(нет фото, короткое описание, нет цены/города, телефон в заголовке и т.д.).
На карточке объявления показываются первые 2 warnings.

## Empty states

- Нет объявлений: «У вас пока нет объявлений» + CTA «Подать объявление»
- Нет заявок: «Заявок пока нет»
- Профиль не заполнен: предупреждение в header + блок improvements
