# Seller Trust (MVP)

Базовый слой доверия продавца. Считается **на лету**, без новых полей в БД и без migration.  
Это **не** верификация документов и **не** отзывы.

## Helper

Файл: `src/lib/trust/seller-trust.ts`

| Функция | Назначение |
|---------|------------|
| `calculateSellerTrust(seller)` | score, level, signals, warnings, improvements |
| `calculateSellerTrustScore(seller)` | число 0–100 |
| `getSellerTrustLevel(score)` | trusted / normal / incomplete |
| `getSellerTrustSignals(seller)` | текстовые сигналы |
| `getSellerTrustWarnings(seller)` | пробелы профиля |

Helper **не делает Prisma queries** — только анализирует переданный объект.

## Как считается score

База: **50**, затем плюсы/минусы:

| Фактор | Баллы |
|--------|-------|
| Телефон подтверждён (`phone_verified_at`) | +20 |
| Есть SellerProfile | +15 |
| Есть название/имя | +15 |
| Есть описание | +10 |
| Есть город | +10 |
| Есть активные (PUBLISHED) объявления | +15 |
| Есть логотип/аватар | +10 |
| Аккаунт старше 7 дней | +5 |
| Нет телефона | −20 |
| Нет описания | −10 |
| Нет активных объявлений | −15 |
| Профиль почти пустой | −20 |

Уровни:

| Score | Level | Label | Badge |
|-------|-------|-------|-------|
| 80–100 | `trusted` | Профиль вызывает доверие | Профиль заполнен |
| 50–79 | `normal` | Обычный профиль | Обычный профиль |
| 0–49 | `incomplete` | Профиль заполнен не полностью | Нужно больше данных |

## Trust signals

Показываются только если есть данные:

- Телефон подтверждён
- Профиль продавца заполнен
- N активных объявлений
- Продавец размещает объявления в ТутОпт / ТутМаркет / …
- На платформе с {date}
- Онбординг пройден

**Не используется** формулировка «проверенный продавец» в trust-слое.  
Поле `SellerProfile.is_verified` остаётся отдельным legacy-бейджем в UI и не смешивается с trust score.

## Где показывается

1. **`/listings/[id]`** — compact блок в карточке продавца  
2. **`/seller/[id]`** — секция «О продавце» в sidebar  
3. **`/seller/dashboard`** — «Заполненность профиля» + improvements  
4. **`/admin/users`** — маленький trust badge для продавцов  

`/seller/settings` **пока нет** — CTA в dashboard ведёт на создание объявления / публичный профиль; редактирование описания/города — отдельная фаза.

## Почему это не верификация документов

Trust score отражает **заполненность и активность** профиля на платформе:

- подтверждение телефона (OTP)
- наличие описания, города, объявлений

Он **не** означает:

- проверку паспорта / ИНН
- проверку юрлица
- отзывы покупателей
- рейтинг сделок

## Ограничения MVP

- Score не сохраняется в БД
- Нет отзывов / звёзд / рейтинга сделок
- Нет manual admin verification workflow
- Нет истории trust
- Нет жалоб в trust formula
- `/seller/settings` отсутствует
