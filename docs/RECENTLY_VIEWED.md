# Недавно просмотренные объявления (Recently Viewed)

## Как работает история просмотров

1. Пользователь открывает страницу объявления `/listings/[id]`.
2. Если объявление опубликовано (`PUBLISHED`), невидимый клиентский компонент
   `RecentlyViewedTracker` сохраняет безопасный минимум данных объявления
   в `localStorage` браузера (на mount, один раз за визит страницы).
3. На `/buyer/dashboard` клиентский блок «Недавно просмотренные»
   (`RecentlyViewedPanel`) читает историю из `localStorage` и показывает
   последние 10 объявлений: фото, направление, цену, город, время просмотра.
4. Из блока можно открыть объявление, удалить один элемент или очистить
   всю историю (с подтверждением через `confirm()`).

Ключевые файлы:

- `src/lib/recently-viewed/local-recently-viewed.ts` — helper для `localStorage`
- `src/components/listings/RecentlyViewedTracker.tsx` — трекер на странице объявления
- `src/components/buyer/RecentlyViewedPanel.tsx` — блок на buyer dashboard

Ключ хранения: `tutopt.recentlyViewedListings.v1`

Примечание: серверная запись просмотров (`user_listing_views` через
`recordListingView`) продолжает работать для авторизованных пользователей —
она осталась как источник данных для будущих серверных фич (рекомендации,
cross-device история). UI-блок на dashboard теперь работает от `localStorage`,
поэтому история доступна и гостям, и её можно чистить локально.

## Почему пока localStorage, а не БД

- Не нужны изменения Prisma schema и миграции.
- История доступна и гостям, без логина.
- Просмотры — не критичные данные: потеря при смене браузера допустима для MVP.
- Пользователь полностью контролирует свою историю (удаление, очистка) без
  запросов к серверу.

## Какие данные сохраняются

Для каждого объявления:

- `id` — id объявления
- `title` — заголовок
- `url` — ссылка вида `/listings/[id]`
- `vertical` — направление (OPT / MARKET / SERVICES / CARGO)
- `price`, `currency` — цена, как она показывается публично
- `city` — город объявления
- `imageUrl` — URL первого фото
- `viewedAt` — время просмотра (ISO string)

Сохраняются только опубликованные объявления. Черновики, объявления на
модерации и отклонённые (даже если их видит владелец/админ) в историю
не попадают.

## Какие данные запрещено сохранять

- телефон и email (продавца или покупателя)
- данные заявок (lead)
- приватные данные seller profile (whatsapp, telegram, contact_phone)
- внутренние admin-поля (статусы блокировок, причины и т.п.)
- любые токены и идентификаторы сессий

## Лимиты и поведение

- Максимум 30 объявлений в истории (старые вытесняются).
- Повторный просмотр не создаёт дубликат: запись поднимается наверх,
  `viewedAt` обновляется.
- Битый JSON в `localStorage` не ломает страницу — история просто считается
  пустой.
- Ошибки записи (private mode, quota) молча игнорируются.

## Analytics

События (без title, seller info и данных пользователя):

- `recently_viewed_open` — переход к объявлению из блока (params: `vertical`, `has_price`)
- `recently_viewed_remove` — удаление одного элемента (params: `vertical`, `has_price`)
- `recently_viewed_clear` — очистка всей истории

Отдельного события на сам просмотр объявления нет — используется уже
существующее `listing_view`.
