# Admin Dashboard (MVP)

Главная страница админки `/admin` — операционные показатели платформы без тяжёлой аналитики.

## Показатели

Первый ряд карточек (виден ADMIN и MODERATOR):

| Карточка | Что считает | Ссылка |
|----------|-------------|--------|
| Объявления на модерации | listings `PENDING_MODERATION` | `/admin/moderation/listings` |
| Новые жалобы | reports `OPEN` | `/admin/reports` |
| Всего объявлений | все listings | — |
| Активные объявления | listings `PUBLISHED` | — |

Второй ряд (только ADMIN):

| Карточка | Что считает | Ссылка |
|----------|-------------|--------|
| Пользователи | все users | `/admin/users` |
| Продавцы | seller profiles | — |
| Заблокированные | users c `is_blocked` или `blocked_at` | `/admin/users` |
| Заявки | все leads + за 7 дней в подписи | — |
| Действия за 7 дней | audit logs за 7 дней | `/admin/audit` |

## Блоки

- **Быстрые действия** — модерация, жалобы, пользователи (ADMIN), журнал (ADMIN), «На сайт».
- **По направлениям** — для OPT / MARKET / SERVICES / CARGO: всего, активных, на модерации. Клик ведёт на модерацию с фильтром по vertical.
- **Последние события** (только ADMIN) — последние 8 записей audit log: label действия, тип объекта, актор, дата. Metadata не показывается. Ссылка «Открыть журнал действий». Пустое состояние: «Журнал действий пока пуст».

## Роли и доступ

- guest → redirect на login
- BUYER / SELLER → redirect на `/`
- MODERATOR → видит модерационные карточки, направления и быстрые действия (без пользователей и журнала)
- ADMIN → видит всё

## Используемые модели

`Listing`, `Report`, `User`, `SellerProfile`, `Lead`, `AuditLog`. Prisma schema не менялась, миграций нет.

## Условные блоки

- Второй ряд карточек, «Последние события», ссылки на `/admin/users` и `/admin/audit` — только для ADMIN.
- Пустая база не ломает страницу: все счётчики показывают 0, у журнала есть empty state.

## Почему нет сложной аналитики в MVP

Дашборд отвечает на операционный вопрос «что требует внимания сейчас»: очередь модерации,
жалобы, блокировки. Графики и тренды требуют агрегационных таблиц или chart-библиотек —
это лишние зависимости и сложность до появления реального объёма данных.
