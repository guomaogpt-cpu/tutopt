# Публичная страница продавца (seller storefront)

Публичная страница продавца находится по адресу `/seller/[id]`, где `[id]` — id или slug
профиля продавца. Страница работает как мини-витрина: шапка продавца, trust-сигналы,
статистика и каталог активных объявлений.

## Структура страницы

1. **Хлебные крошки** — Главная / Каталог / имя продавца.
2. **Шапка продавца (sidebar)**:
   - логотип или инициалы;
   - роль/тип по основному направлению («Поставщик», «Продавец», «Исполнитель», «Перевозчик»);
   - бейдж «Проверен» — только если `is_verified` выставлен вручную админом;
   - trust badge из seller trust helper;
   - название компании и контактное лицо;
   - направления с количеством объявлений;
   - город/регион, дата регистрации, число активных объявлений;
   - CTA «Связаться» (якорь на контакты) и «Смотреть объявления» (якорь на каталог, mobile).
3. **Блок «Доверие к продавцу»** — trust level label и честные trust-сигналы.
4. **Блок «О продавце»** — описание профиля, контактное лицо, город/регион.
5. **Статистика продавца** — компактные карточки.
6. **Каталог «Объявления продавца»** — фильтр по vertical, поиск, категория, сортировка.
7. **CTA-подсказка** — «Чтобы связаться с продавцом, откройте интересующее объявление
   и отправьте заявку.»
8. **Жалоба** — вторичная кнопка «Пожаловаться на продавца» внизу sidebar
   (существующий `ReportDialog`).

## Какие данные показываются

- Название компании, имя контактного лица, логотип/аватар.
- Город и регион.
- Дата регистрации на платформе («На платформе с»).
- Описание профиля (plain text, без `dangerouslySetInnerHTML`).
- Количество активных (PUBLISHED) объявлений — всего и по направлениям.
- Контакты (телефон/email/WhatsApp/Telegram/сайт) — только для авторизованных
  пользователей, это существующее поведение.
- Trust badge и trust-сигналы: «Телефон подтверждён», «Профиль заполнен»,
  «Есть активные объявления» и т.п.

## Какие данные НЕ показываются

- Email/телефон пользователя из auth (показываются только контакты,
  которые продавец сам указал в профиле, и только авторизованным).
- Внутренние admin-поля: `blocked_at`, ограничения, статусы модерации.
- Отклонённые, pending, draft объявления — в каталог попадают только
  объявления со статусом `PUBLISHED`.
- Надпись «проверенный продавец» без ручной верификации: бейдж «Проверен»
  показывается только при `is_verified = true`.

## Trust signals

Считаются через `calculateSellerTrust` (`src/lib/trust/seller-trust.ts`) на сервере
по данным профиля: заполненность профиля, подтверждённый телефон, наличие описания,
города, логотипа, активных объявлений и направлений. На странице показываются:

- trust badge (уровень: trusted / normal / incomplete) в шапке;
- текстовый label уровня и список сигналов в блоке «Доверие к продавцу».

Если сигналов нет — «Продавец пока не заполнил подробную информацию.»
Если нет описания — «Описание продавца пока не заполнено.»

## Статистика продавца

Компактные карточки над каталогом:

- Активные объявления (только PUBLISHED);
- Направления (количество verticals с объявлениями);
- На платформе с (дата регистрации пользователя);
- Карточка на каждый vertical, где есть объявления
  (ТутОпт / ТутМаркет / ТутУслуги / ТутКарго) с количеством.

## Каталог и фильтр по vertical

- Выводятся только `PUBLISHED` объявления продавца, сортировка по `published_at desc`.
- Фильтр по vertical — chips «Все / ТутОпт / ТутМаркет / ТутУслуги / ТутКарго»
  с количеством; активный vertical передаётся через query-параметр `?vertical=`
  и применяется на сервере.
- Дополнительно на клиенте: поиск по названию, фильтр по категории,
  сортировка (новые / дешевле / дороже).
- Показываются первые 12 карточек, дальше кнопка «Показать ещё» (+12).
  Серверная pagination — later.
- Карточки — универсальный `ListingCard` (variant="catalog") с favorite-кнопкой.

## Почему связь только через объявления

Прямой формы «написать продавцу» на витрине нет: заявка привязана к конкретному
объявлению, чтобы продавец видел контекст (товар/услуга, цена, направление),
а платформа могла применять анти-спам и ограничения на уровне lead submit.
На витрине показывается подсказка открыть объявление и отправить заявку.

## Analytics

- `seller_profile_view` — при открытии страницы
  (`seller_has_profile`, `vertical` = primary, `listing_count_bucket`).
- `seller_listing_click` — клик по карточке объявления в каталоге продавца
  (`vertical`, `listing_count_bucket`).
- `seller_vertical_filter_click` — клик по chip фильтра (`vertical`, для «Все» без vertical).

Не отправляются: имя продавца, phone/email, данные пользователя.
`listing_count_bucket`: `0`, `1-3`, `4-10`, `11+`.

## SEO

`generateMetadata`:

- title: имя продавца + брендовое направление (например, «… — поставщик на ТутОпт»);
- description: описание профиля (если оно не короче 40 символов), обрезанное
  до ~155 символов, иначе сгенерированный fallback с городом и количеством объявлений;
- обёрнуто в try/catch с fallback-метаданными — build не падает при недоступной БД.

## Заблокированные продавцы

Отдельной публичной политики скрытия профиля заблокированного продавца нет:
блокировка ограничивает действия самого пользователя (создание объявлений, заявки),
но старые опубликованные объявления и профиль остаются видимыми. Публично статус
«заблокирован» нигде не показывается.
