# Platform Verticals Strategy

Продуктовая стратегия расширения Tutopt в мульти-направленную classified-платформу.

**Статус:** стратегия / Phase 0  
**Код / UI / Prisma / migration:** не менять в рамках этого документа  
**Цель:** зафиксировать архитектуру направлений (vertical), naming, навигацию, SEO и поэтапный план без ломки текущего B2B-ядра.

---

## 1. Контекст

Сейчас **Tutopt** — B2B classified для оптовых товаров: поставщики, MOQ, партии, заявки покупателей.

Новая стратегия — сделать платформу **шире по охвату** (сопоставимо с Lalafo), но **структурированнее и сильнее по UX/SEO**.

### Главная проблема Lalafo

- Всё лежит в **одной общей ленте**.
- Товары, услуги, розница, опт, авто, работа, грузоперевозки **смешаны**.
- Пользователю сложно сразу попасть в **нужный контекст намерения**.
- SEO-структура слабее: категории перегружены, разные сценарии конкурируют за одни и те же URL и фильтры.

### Наша идея

Ввести **верхний уровень направлений платформы** (vertical), а уже внутри направления — свои категории, фильтры, формы и SEO.

Не «мусорка с табами», а **несколько крупных продуктов в одной экосистеме** с общим брендом и аккаунтом.

## 3. Naming

### Основной вариант

| Направление | Название |
|-------------|----------|
| Опт (B2B) | **ТутОпт** |
| Розница / classified | **ТутМаркет** |
| Услуги | **ТутУслуги** |
| Логистика | **ТутКарго** |

### Почему «ТутМаркет» лучше, чем «ТутРозница»

- **короче** и легче произносится;
- звучит **шире**: не только «розничная полка», а площадка объявлений;
- включает **розницу + магазины + частные объявления**;
- лучше для **бренда** (привычный marketplace-язык, меньше «узкая ниша»).

**Для ТутМаркет:**

- ТутМаркет *(рекомендуется)*
- ТутТовар
- ТутБазар
- ТутРозница

**Для ТутУслуги:**

- ТутУслуги *(рекомендуется)*
- ТутМастер
- ТутСервис

**Для ТутКарго:**

- ТутКарго *(рекомендуется)*
- ТутЛогистика
- ТутГруз

### Рекомендация по названиям

Зафиксировать линейку:

**ТутОпт · ТутМаркет · ТутУслуги · ТутКарго**

Префикс «Тут» даёт единый зонтичный бренд; у каждого направления — узнаваемый суффикс.

## 4. Роль vertical

**vertical** = верхний уровень направления платформы.

### Возможные значения (код)

```text
OPT
MARKET
SERVICES
CARGO
```

| Код | Продуктовое имя |
|-----|-----------------|
| `OPT` | ТутОпт |
| `MARKET` | ТутМаркет |
| `SERVICES` | ТутУслуги |
| `CARGO` | ТутКарго |

### Будущая структура данных / URL

```text
vertical → category → subcategory → listing
```

**Пример (ТутОпт):**

```text
OPT → Продукты питания → Фрукты → Яблоки
```

Категории **не шарятся** между вертикалями на старте: у каждого направления своё дерево. Это упрощает фильтры, модерацию и SEO.

---

## 5. Почему лучше заложить сейчас

Сайт пока **почти пустой** (мало объявлений, слабый накопленный SEO). Сейчас — лучший момент ввести vertical.

Если отложить и делать позже, появятся проблемы:

- миграция старых объявлений;
- путаница категорий («куда отнести объявление?»);
- смена URL;
- цепочки редиректов;
- потеря / размытие SEO;
- сложная модерация смешанного контента;
- пользователи уже привыкнут к плоской структуре.

**Сейчас изменение относительно безопасное:** бэкфилл «всё = OPT» короткий, редиректы минимальны, привычка ещё не закрепилась.

## 6. Навигация

### Вариант A — один домен + path *(рекомендуется для MVP)*

```text
/opt
/market
/services
/cargo
```

### Вариант B — subdomains

```text
opt.tutopt.kg
market.tutopt.kg
services.tutopt.kg
cargo.tutopt.kg
```

### Вариант C — отдельные домены

```text
tutopt.kg
tutmarket.kg
tutuslugi.kg
tutcargo.kg
```

### Сравнение

| | Плюсы | Минусы |
|---|--------|--------|
| **A. Path** | Один deploy, одна auth/БД, проще SEO на старте, проще поддержка, можно позже уйти в B | Чуть длиннее URL |
| **B. Subdomain** | Сильный визуальный split бренда | Cookies/CORS, сложнее Next/SSR/auth, больше ops |
| **C. Отдельные домены** | Максимальная изоляция бренда | 4 продукта по факту: маркетинг, SSL, SEO, поддержка — дорого |

### Рекомендация для MVP

Использовать **один домен и path:**

```text
/opt
/market
/services
/cargo
```

**Причины:** проще разработка; проще SEO на старте; единая авторизация; единая база; проще поддержка; позже возможен переход на subdomains с 301-картой.

## 7. Header UX

### Будущий header

Верхний переключатель направлений:

```text
[ТутОпт] [ТутМаркет] [ТутУслуги] [ТутКарго]
```

### Поведение

- выбранное направление **подсвечено**;
- **поиск** работает внутри выбранного направления;
- **категории** меняются под выбранное направление;
- кнопка **«Подать объявление»** понимает выбранное направление (контекст vertical).

### Для MVP (после Phase 0)

- добавить переключатель в header;
- по умолчанию активен **ТутОпт**;
- остальные направления показывать как **«Скоро»** или вести на **заглушку** (`/market`, `/services`, `/cargo`).

*Реализация UI — в следующих фазах; этот документ код не меняет.*

## 10. SEO-архитектура

### Будущие URL (примеры)

**ТутОпт:**

```text
/opt/produkty-pitaniya/frukty/yabloki/bishkek
/opt/stroymaterialy/cement/bishkek
```

**ТутМаркет:**

```text
/market/telefony/iphone/bishkek
/market/odezhda/muzhskaya-odezhda/osh
```

**ТутУслуги:**

```text
/services/remont/santehniki/bishkek
/services/obuchenie/angliyskiy-yazyk/osh
```

**ТутКарго:**

```text
/cargo/kitay-kyrgyzstan/dostavka-gruzov
/cargo/gruzoperevozki/bishkek-osh
```

### SEO-элементы

| Элемент | Назначение |
|---------|------------|
| **ЧПУ URL** | Читаемые path без лишних query |
| **slug** | Для vertical (в path), category, city — стабильные латиница/транслит |
| **title** | Уникальный, с брендом вертикали |
| **meta description** | 1–2 предложения + CTA |
| **H1** | Соответствует намерению страницы (категория / услуга / маршрут) |
| **breadcrumbs** | Вертикаль → категория → … → город |
| **category SEO text** | Короткий редакторский блок на категорийных лендингах |
| **canonical** | Self на публичных страницах; аккуратная пагинация |
| **sitemap.xml** | По вертикалям + listings чанками |
| **robots.txt** | Allow публичное; Disallow admin/api/draft |
| **JSON-LD** | Product/Offer, Service, ItemList, Organization |
| **OpenGraph** | Превью для шаринга карточек и категорий |
| **city landing** | `…/{city}` |
| **category + city** | Основной SEO-юнит роста |

### Примеры title

```text
Купить яблоки оптом в Бишкеке — цены поставщиков | ТутОпт
iPhone в Бишкеке — новые и б/у объявления | ТутМаркет
Сантехники в Бишкеке — услуги мастеров | ТутУслуги
```

## 12. Влияние на существующий сайт

Текущий Tutopt можно считать **`vertical = OPT`**.

### Текущие объявления

- будут считаться **OPT**;
- текущие категории можно привязать к **OPT**;
- текущая главная остаётся рабочей.

### Что изменится со временем

- создание объявления будет выбирать **vertical**;
- поиск будет **фильтровать по vertical**;
- категории будут **зависеть от vertical**;
- URL со временем станут **SEO-friendly** (`/opt/...` и т.д.).

Цель миграции — **не ломать** рабочий продуктовый контур опта.

## 14. Риски

| Риск | Суть |
|------|------|
| Слишком раннее раздувание MVP | Четыре «полноценных» продукта сразу |
| Разные формы | Услуги и карго ≠ карточка товара |
| Сложнее модерация | Разные scam-паттерны и правила |
| SEO требует времени | Хабы без контента не ранжируются |
| Нужны контент и категории | Деревья и SEO-тексты — отдельный объём работ |
| Нельзя ломать ТутОпт | Ядро уже в production |

### Рекомендация по рискам

**Не пытаться сразу сделать 4 полноценных продукта.**  
Сначала заложить **vertical** и продолжать развивать **ТутОпт как ядро**. Остальные направления — stub → soft launch по одному.

## 15. Финальная рекомендация

1. **Идея правильная** — структура направлений сильнее общей ленты Lalafo.  
2. **Сейчас подходящий момент** — мало данных, низкая цена ошибки.  
3. **Делать поэтапно** — по фазам выше, без большого rewrite.  
4. **Сначала vertical-архитектура** — enum + поля + бэкфилл OPT.  
5. **Не переписывать весь сайт сразу.**  
6. **ТутОпт оставить ядром.**  
7. **Остальные направления запускать постепенно** (Маркет / Услуги / Карго — после закладки модели и UI-переключателя).

Naming для продукта: **ТутОпт · ТутМаркет · ТутУслуги · ТутКарго**.  
Routing для MVP: **path** `/opt` · `/market` · `/services` · `/cargo`.  
Главная: пока **ТутОпт**, переключатель направлений — сверху.
