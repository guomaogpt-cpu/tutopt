# Tutopt — Роли пользователей и права доступа

## 1. Обзор модели доступа

Tutopt использует **RBAC** (Role-Based Access Control): каждый пользователь имеет одну основную роль. Дополнительные флаги (верификация, PRO) не заменяют роль, а расширяют возможности в рамках неё.

---

## 2. Роли

### 2.1 Гость (неавторизованный)

**Описание:** любой посетитель сайта без аккаунта.

| Действие | Доступ |
|----------|--------|
| Просмотр каталога, категорий, объявлений | ✅ |
| Поиск и фильтры | ✅ |
| Просмотр профиля продавца (публичная часть) | ✅ |
| Просмотр телефона продавца | ❌ (маскируется) |
| Отправка заявки / сообщения | ❌ |
| Добавление в избранное | ❌ (локально в cookie — опционально) |
| Создание объявлений | ❌ |

**Редирект:** при попытке защищённого действия → `/auth/login?redirect=...`

---

### 2.2 Покупатель (BUYER)

**Описание:** зарегистрированный пользователь, ищущий товары для закупки.

**Регистрация:** email или телефон + пароль. Роль по умолчанию при регистрации.

| Действие | Доступ |
|----------|--------|
| Всё, что доступно гостю | ✅ |
| Просмотр полных контактов продавца | ✅ |
| Отправка заявки на опт (lead) | ✅ |
| Избранное (синхронизация с аккаунтом) | ✅ |
| История заявок | ✅ |
| Жалоба на объявление | ✅ |
| Создание объявлений | ❌ |
| Редактирование чужих объявлений | ❌ |

**Профиль покупателя:** имя, телефон, email, город, аватар (опционально).

---

### 2.3 Продавец (SELLER)

**Описание:** компания или ИП, публикующая оптовые объявления.

**Получение роли:** при регистрации выбор «Я продавец» или апгрейд в личном кабинете + заполнение профиля компании.

| Действие | Доступ |
|----------|--------|
| Всё, что доступно покупателю | ✅ |
| Создание / редактирование своих объявлений | ✅ |
| Управление профилем компании | ✅ |
| Просмотр статистики своих объявлений | ✅ |
| Просмотр входящих заявок (leads) | ✅ |
| Ответ на заявки (статус: просмотрено) | ✅ |
| Модерация чужих объявлений | ❌ |
| Доступ к админ-панели | ❌ |

**Ограничения MVP:**

- Максимум **10 активных** объявлений на бесплатном тарифе
- Новые объявления проходят **модерацию** перед публикацией
- Неверифицированный продавец: бейдж «Новый продавец», лимит 5 объявлений

**Профиль компании (обязательные поля):**

- Название компании
- Город / регион
- Контактный телефон
- Описание (опционально)
- Логотип (опционально)

**Профиль компании (для верификации):**

- ИНН / регистрационный номер
- Юридический адрес
- Документы (загрузка, видит только модератор)

---

### 2.4 Модератор (MODERATOR)

**Описание:** сотрудник Tutopt, обрабатывающий контент.

| Действие | Доступ |
|----------|--------|
| Очередь модерации объявлений | ✅ |
| Одобрение / отклонение объявлений | ✅ |
| Очередь верификации продавцов | ✅ |
| Просмотр жалоб | ✅ |
| Блокировка объявления / продавца | ✅ |
| Редактирование категорий | ❌ |
| Управление пользователями (роли) | ❌ |
| Системные настройки | ❌ |

**Интерфейс:** `/admin/moderation/*`

---

### 2.5 Администратор (ADMIN)

**Описание:** полный доступ к платформе.

| Действие | Доступ |
|----------|--------|
| Все права модератора | ✅ |
| Управление категориями | ✅ |
| Управление пользователями и ролями | ✅ |
| SEO-страницы, баннеры | ✅ |
| Аналитика и отчёты | ✅ |
| Системные настройки | ✅ |
| Удаление любых данных | ✅ |

**Интерфейс:** `/admin/*`

---

## 3. Дополнительные статусы (не роли)

| Статус | Описание | Влияние |
|--------|----------|---------|
| `email_verified` | Email подтверждён | Обязательно для публикации объявлений |
| `phone_verified` | Телефон подтверждён (SMS) | Открывает контакты, снижает спам |
| `seller_verified` | Документы проверены модератором | Бейдж «Проверенный», приоритет в выдаче |
| `seller_pro` | Платная подписка (фаза 2) | Больше объявлений, PRO-бейдж |
| `is_blocked` | Аккаунт заблокирован | Логин запрещён, объявления скрыты |

## 5. Аутентификация и сессии

### 5.1 Методы входа (MVP)

- Email + пароль
- Телефон + пароль (формат +996...)
- Восстановление пароля по email

### 5.2 Сессии

- JWT в httpOnly cookie или session ID в Redis (фаза 2)
- Срок сессии: 30 дней (remember me) / 24 ч (без remember)
- Принудительный logout при смене пароля и блокировке

### 5.3 Middleware

Next.js middleware проверяет:

- Публичные маршруты — без проверки
- `/account/*`, `/seller/*` — требуется авторизация
- `/admin/*` — роль MODERATOR или ADMIN

---

## 6. Аудит действий

Логируются (таблица `audit_logs`):

- Смена роли пользователя
- Одобрение / отклонение объявления
- Блокировка пользователя
- Верификация продавца

Поля: `actor_id`, `action`, `entity_type`, `entity_id`, `metadata`, `created_at`.
