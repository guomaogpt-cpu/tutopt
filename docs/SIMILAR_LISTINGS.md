# Похожие объявления (Similar Listings)

## Где показываются

На странице объявления `/listings/[id]`, в самом низу — после характеристик,
описания и формы заявки (блок не конкурирует с CTA заявки).

Компонент: `src/components/listings/SimilarListings.tsx` (client, получает
уже сериализованные `ListingCardData`).
Подбор: `getSimilarListings` в
`src/features/listings/lib/listing-detail-data.ts` (серверный helper).

## Как выбираются похожие объявления

Тиерный подбор — до трёх лёгких запросов, каждый добирает только недостающее
количество (`take = remaining`):

1. **Та же категория + то же направление** (category_id + vertical)
2. **То же направление** (vertical)
3. **Тот же город** (city_id, любое направление) — только если у объявления
   указан город

Внутри каждого тира сортировка по `published_at desc` (свежие первыми).
Уже добранные объявления исключаются из следующих тиров (`id notIn`).

## Что исключается

- текущее объявление
- все статусы кроме `PUBLISHED` (DRAFT / PENDING_MODERATION / REJECTED /
  ARCHIVED не показываются)

## Лимит

Максимум **4 объявления**. Если после всех тиров ничего не найдено —
блок не рендерится вовсе (нет пустого заголовка «Похожие объявления»),
страница объявления выглядит аккуратно.

## Карточки

Используется универсальная `ListingCard` (variant="catalog"): фото или
placeholder «Нет фото», vertical badge (существующие лейблы/цвета,
без vertical-theme), title, цена («Цена не указана» / «Цена договорная»
через общий форматтер), город/MOQ/бренд по vertical, favorite button.
Decimal/Date сериализуются через существующий `serializeListingCards`
до передачи в client component.

## Analytics

Событие `similar_listing_click` (клик по ссылке карточки, не по избранному):

- `source_vertical` — направление текущего объявления
- `target_vertical` — направление объявления, на которое кликнули
- `same_category` — совпадает ли категория с текущим объявлением

Не отправляются: title, имя продавца, телефон/email, данные пользователя.

## Что позже

- рекомендации на основе истории просмотров (recently viewed / user_listing_views)
- похожие по цене (диапазон ±20%)
- приоритет города в ранжировании, а не отдельным тиром
- ML-рекомендации
- персонализация по активности покупателя (избранное, заявки)
