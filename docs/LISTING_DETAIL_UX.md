# Страница объявления /listings/[id] — UX (Phase 27)

## Структура страницы

Серверный компонент `src/app/listings/[id]/page.tsx` собирает данные и рендерит:

1. **Хлебные крошки** — Главная → направление → категория → объявление.
2. **Верхний блок** — vertical badge (Опт / Маркет / Услуга / Карго),
   статус (только владельцу и админам, для непубличных статусов),
   бейдж «Ваше объявление» владельцу, title, мета-строка:
   категория · город · дата публикации.
3. **Compact admin block** («Для модерации») — только ADMIN/MODERATOR:
   статус, risk badge из listing-quality helper, ссылка в `/admin/moderation/listings`.
4. **Галерея** (`ListingGallery`) — первое фото крупно, миниатюры при
   нескольких фото, placeholder «Фото не добавлено» без фото.
5. **Contact card** (`ListingContactCard`) — цена (или «Цена не указана» /
   «Цена договорная» для SERVICES/CARGO), MOQ/город/бренд/наличие по vertical,
   primary CTA (скролл к форме), избранное, контакты (телефон/WhatsApp/Telegram
   только авторизованным).
6. **Seller card** (`ListingSellerCard`) — см. ниже.
7. **Характеристики** (`ListingCharacteristics`) — только заполненные поля:
   направление, категория, город, цена, MOQ, единица, наличие, бренд,
   дата публикации. Технические id не выводятся.
8. **Описание** (`ListingDescription`) — переносы строк через
   `whitespace-pre-wrap` (без dangerouslySetInnerHTML), длинный текст
   сворачивается, пустое описание → «Описание не указано».
9. **Vertical hint** (`ListingVerticalHint`) — подсказка перед формой заявки
   (не показывается владельцу):
   - OPT: уточните минимальную партию, наличие и условия отгрузки
   - MARKET: уточните состояние, комплектацию и доставку
   - SERVICES: опишите задачу, сроки и место выполнения
   - CARGO: укажите маршрут, тип груза, вес/объём и сроки
10. **Lead form** (`ListingLeadForm`) — см. ниже.
11. **Listing quality hints** — только владельцу, под формой: оценка качества
    объявления и советы по улучшению (существующий helper `listing-quality`).
12. **Похожие объявления** (`SimilarListings`).

На mobile contact/seller card показываются после галереи; на desktop —
в sticky-сайдбаре.

## Seller card

- имя/название компании, аватар/логотип
- роль по vertical (Поставщик / Продавец / Исполнитель / Перевозчик)
- честные trust-сигналы из seller-trust helper («Телефон подтверждён»,
  «Профиль заполнен» и т.п.) — «Проверен» показывается только при
  `is_verified` (ручная проверка)
- количество опубликованных объявлений, город, дата «На платформе с»
  (передаётся уже отформатированной строкой — Date не пересекает границу
  client component)
- CTA (скролл к форме) + «Профиль продавца» → /seller/[id]
- маленькая вторичная кнопка «Пожаловаться» (не конкурирует с CTA)

## CTA / Lead form состояния

| Состояние | Что видит пользователь |
|-----------|------------------------|
| guest | Заголовок формы + кнопка «Войдите, чтобы отправить заявку» → `/login?next=<current>` |
| авторизован | Форма заявки + подпись «Отправьте заявку — продавец увидит её в своём кабинете.» |
| restricted (blocked / lead_restricted_at) | Карточка с сообщением «Отправка заявок временно ограничена.» / «Аккаунт заблокирован…» вместо формы (серверная блокировка в lead API осталась из Phase 19) |
| owner | «Это ваше объявление — заявки от клиентов появятся в разделе „Заявки“» + ссылка на /seller/leads + quality hints |

Тексты формы, шаблоны сообщений и подписи CTA настраиваются per-vertical в
`lead-form-config.ts`.

## Analytics

События (`src/lib/analytics/events.ts`, `trackListingDetailAction`):

- `listing_contact_cta_click` — клик по CTA в contact card или seller card
- `listing_seller_profile_click` — переход в профиль продавца
- `listing_report_click` — открытие диалога жалобы

Params: `vertical`, `has_price`, `is_own_listing`.
Не отправляются: title, имя продавца, телефон/email, текст заявки.
Просмотр объявления — существующий `listing_view`, не дублируется.

## Примечания

- Кнопки «Добавить к сравнению» нет: функциональность сравнения объявлений
  в кодовой базе отсутствует, нерабочие кнопки не добавляются.
- Кнопки «Редактировать» у владельца нет: route редактирования объявления
  ещё не существует.
- SEO metadata: title + цена + Tutopt, description ≤155 символов из описания
  с fallback, обёрнуто в try/catch, non-published — noIndex.
