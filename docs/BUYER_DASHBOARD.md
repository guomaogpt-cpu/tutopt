# Buyer Dashboard (MVP)

Кабинет покупателя `/buyer/dashboard`. Доступ: любой авторизованный пользователь
(guest редиректится на login). Навигация: для BUYER «Кабинет покупателя» в header-меню
и мобильном drawer ведёт сюда; SELLER сохраняет `/seller/dashboard`, ADMIN/MODERATOR — `/admin`.

## Блоки

1. **Header** — имя пользователя, badge «Покупатель», «Телефон подтверждён» (если есть
   `phone_verified_at`), дата регистрации, кнопки «Найти объявления» → `/listings` и
   «Избранное» → `/favorites`.
2. **Предупреждение об ограничениях** — если пользователь заблокирован: «Аккаунт ограничен.
   Некоторые действия недоступны.»; если ограничены только заявки: «Отправка заявок временно
   ограничена.» Это только отображение — сам запрет работает в lead submit API (Phase 19).
3. **Моя активность** — карточки: избранные объявления, отправленные заявки, заявки за
   7 дней, новые уведомления.
4. **Направления интереса** — verticals (ТутОпт/ТутМаркет/ТутУслуги/ТутКарго), где покупатель
   добавлял избранное или отправлял заявки; чипы-ссылки на разделы. Скрывается, если активности нет.
5. **Быстрый поиск** — Опт → `/opt`, Маркет → `/market`, Услуги → `/services`,
   Карго → `/cargo`, Все объявления → `/listings`. Без формы и фильтров.
6. **Мои заявки** — последние 10 (с общим счётчиком): дата, объявление с фото, продавец
   (ссылка на публичный профиль), vertical badge, статус заявки, ссылка на объявление.
7. **Избранные товары** — последние 6 карточек `ListingCard` (фото, title, цена, город,
   vertical), ссылка «Смотреть всё» → `/favorites`.
8. **Недавно просмотренные** — существующий блок истории просмотров.
9. **Быстрые действия** — каталог, избранное, уведомления, якорь «Мои заявки».

## Данные

- `getBuyerDashboardData` (`src/features/buyer/lib/buyer-dashboard-data.ts`):
  избранное (`getUserFavoriteListings`), заявки покупателя (`getBuyerLeads`, по `buyer_id`),
  недавние просмотры, verticals избранного.
- `getUnreadNotificationCount` — счётчик уведомлений.
- `isUserBlocked` / `canUserSendLeads` (`src/lib/security/user-restrictions.ts`) — ограничения.

Prisma schema не менялась. Лишние персональные данные продавца не выводятся —
только `company_name` и ссылка на публичный профиль (как в `/seller/leads` и раньше).

## Empty states

- Нет заявок: «Вы ещё не отправляли заявки» + CTA «Перейти в каталог»
- Нет избранного: «В избранном пока пусто» + CTA «Открыть каталог»
- Нет активности: блок «Направления интереса» скрыт, счётчики показывают 0
