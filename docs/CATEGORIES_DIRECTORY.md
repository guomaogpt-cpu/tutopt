# Каталог категорий `/categories`

## Зачем вынесли категории с главной

Главная `/` позиционирует платформу целиком (опт, объявления, услуги, карго)
и должна оставаться лёгкой. Большой блок «Популярные категории» убран с
главной, чтобы не конкурировать с hero и блоком направлений.

Категории по-прежнему нужны для быстрого входа в каталог — для этого есть
отдельная страница **`/categories`**.

## Где находится страница

- Route: `src/app/categories/page.tsx`
- UI (фильтр + карточки): `src/components/categories/CategoriesDirectory.tsx`
- Данные: `src/features/home/lib/categories-page-data.ts`
- Иконки / склонения: `src/features/home/lib/category-icons.ts`

Страница публичная (guest / user / admin).

## Группировка по vertical

У модели `Category` есть поле `vertical` (`OPT` | `MARKET` | `SERVICES` | `CARGO`).

На `/categories`:
- загружаются **корневые** активные категории **всех** направлений;
- при фильтре «Все» категории сгруппированы по секциям:
  - ТутОпт
  - Объявления (не «ТутМаркет»)
  - ТутУслуги
  - ТутКарго
- pill-tabs сверху фильтруют client-side без дополнительных запросов к БД
- есть client-side поиск по названию / slug

## Ссылки в `/listings`

Карточка категории ведёт на:

```
/listings?vertical=<VERTICAL>&category=<id>
```

Формат `category=<id>` совпадает с фильтрами каталога
(`src/features/listings/lib/listings-catalog.ts`). Параметр `vertical`
открывает нужный раздел каталога.

На главной вторичная ссылка «Смотреть все категории» → `/categories`
(в блоке направлений, без возврата большого блока популярных категорий).

## Counts

Да: количество опубликованных (не истёкших) объявлений в дереве категории
считается через один `groupBy` по `category_id` + суммирование потомков.
Отдельных тяжёлых запросов на каждую категорию нет.
