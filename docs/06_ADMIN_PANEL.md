# Tutopt — Админ-панель

## 1. Обзор

Админ-панель — внутренний инструмент команды Tutopt для модерации контента, управления каталогом и мониторинга платформы.

- **URL:** `/admin`
- **Доступ:** роли MODERATOR и ADMIN
- **UI:** Shadcn UI + Tailwind, тёмная/светлая тема
- **Layout:** фиксированный sidebar + header + content

## 4. Модерация объявлений `/admin/moderation/listings`

### 4.1 Список

**Фильтры:**

- Статус: PENDING (default), REJECTED, ALL
- Категория
- Дата подачи
- Продавец (поиск)

**Колонки таблицы:**

| Колонка    | Описание                  |
| ---------- | ------------------------- |
| Превью     | Первое фото               |
| Название   | Ссылка на preview         |
| Продавец   | Компания + verified badge |
| Категория  |                           |
| Цена / MOQ |                           |
| Дата       | created_at                |
| Действия   | Approve / Reject          |

**Bulk actions:** одобрить выбранные (ADMIN only)

### 4.2 Детальный просмотр (drawer / modal)

- Полная галерея
- Описание и атрибуты
- История: кто создал, сколько раз редактировалось
- Кнопки: **Одобрить**, **Отклонить** (с обязательной причиной), **Заблокировать продавца**

### 4.3 Причины отклонения (preset)

- Некорректные фото
- Запрещённый товар
- Неверная категория
- Дублирование
- Подозрение на мошенничество
- Другое (свободный текст)

### 4.4 SLA

- Целевое время модерации: **< 24 часов**
- Badge «просрочено» если PENDING > 24ч

## 5. Верификация продавцов `/admin/moderation/sellers`

### 5.1 Очередь

Продавцы со статусом документов PENDING.

**Карточка:**

- Данные компании
- Загруженные документы (просмотр в lightbox)
- История объявлений продавца
- Approve / Reject

### 5.2 После одобрения

- `seller_profiles.is_verified = true`
- Email уведомление продавцу
- Бейдж на публичном профиле

## 7. Категории `/admin/categories` (ADMIN)

### 7.1 Дерево

- Drag-and-drop сортировка
- Inline edit названия
- Добавить подкатегорию
- Деактивировать (не удалять если есть объявления)

### 7.2 Форма категории

| Поле        | Описание                    |
| ----------- | --------------------------- |
| name        | Название                    |
| slug        | Auto из name, редактируемый |
| parent      | Select родителя             |
| description | SEO-текст                   |
| icon        | Выбор из набора             |
| sort_order  | Число                       |

## 8. Пользователи `/admin/users` (ADMIN)

### 8.1 Список

**Фильтры:** роль, статус (active/blocked), дата регистрации, поиск по email/phone/name

## 10. SEO-страницы `/admin/seo-pages` (ADMIN)

CRUD для кастомных лендингов (`/p/[slug]` или прямой slug).

Поля: title, meta_title, meta_description, content (WYSIWYG), is_published.

**Use case:** «Оптовая торговля в Бишкеке», «Поставщики продуктов питания КР»

## 13. UX-требования

- Подтверждение деструктивных действий (modal)
- Toast-уведомления об успехе/ошибке
- Keyboard shortcuts: `a` approve, `r` reject в модерации
- Responsive: минимум tablet (768px); mobile — read-only очередь
