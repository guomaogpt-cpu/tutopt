# Architecture Decision Records (ADR)

Зафиксированные архитектурные решения проекта Tutopt.

## Назначение

- Документировать **почему** принято решение, а не только **что** сделано
- Избегать повторных дискуссий по уже решённым вопросам
- Давать контекст новым участникам команды и AI-ассистентам

## Формат ADR

```markdown
# ADR-NNN: Название решения

## Статус

proposed | accepted | deprecated | superseded

## Контекст

...

## Решение

...

## Последствия

### Плюсы

- ...

### Минусы

- ...

## Альтернативы

- ...
```

## Правила

- Именование: `ADR-NNN-short-title.md`
- Новое решение не отменяет старое молча — помечать `superseded by ADR-XXX`
- Связанные ADR указывать в секции «Контекст»
