# Tasks

Рабочие задачи и чеклисты разработки Tutopt.

## Назначение

- Декомпозиция эпиков на конкретные задачи
- Чеклисты перед релизом и code review
- Отслеживание прогресса по MVP

## Формат файла задачи

```markdown
# TASK-001: Название

## Статус
pending | in_progress | done

## Описание
...

## Критерии готовности
- [ ] ...

## Связанные документы
- docs/...
```

## Правила

- Один файл = одна задача или один спринт
- Именование: `TASK-NNN-short-name.md` или `sprint-NN.md`
- Закрытые задачи не удалять — переносить статус в `done`
