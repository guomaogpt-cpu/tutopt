# Tasks — Engineering Backlog Tutopt

Полный backlog MVP разбит по спринтам.

## Структура

- `00_SPRINT_OVERVIEW.md` — обзор всех спринтов
- `sprint-XX-name/` — задачи спринта в формате `SXX-NNN.md`

## Формат задачи

Каждый файл содержит:
- ID, название, цель, контекст
- Scope и ограничения
- Критерии готовности
- Команды проверки
- Prompt для Cursor
- Правило commit

## Правила

- Выполнять задачи последовательно внутри спринта
- Не смешивать несколько задач в одном PR без необходимости
- Tutopt — B2B classified, не e-commerce

## Статусы (вручную в задаче или трекере)

`pending` → `in_progress` → `done`
